Name

about:pattern-matching

See also

case-match ecase-match if-match when-match

Description

Inspired by the Common Lisp library optima.

The simplest pattern is a constant, such as a number, string or 'expression. It matches anything that is equal to it.

user > (if-match 123 (+ 100 23) true)
it: true
user > (var expr '(1 2 3))
it: expr
user > (if-match '(1 2 3) expr true)
it: true

The symbol pattern is usually found as a subpattern of a structural pattern. The symbol names a variable that will be declared and bound when the pattern matches.

user > (var expr '(1 2 3))
it: expr
user > (if-match (list a b c) expr "a=${a} b=${b} c=${c}")
it: "a=1 b=2 c=3"
user > (if-match (list 1 2 c) expr "c=${c}")
it: "c=3"

Patterns can be combined with and, or and not.

user > (if-match (and (satisfies integer?) (satisfies odd?)) "a" true)
it: null
user > (if-match (and (satisfies integer?) (satisfies odd?)) 123 true)
it: true
user > (if-match (and (satisfies integer?) (satisfies odd?) x) 123 x)
it: 123

The property pattern tests one or more properties of a prototype object.

user > (var obj (new :name "teun" :city "leiden" :country "netherlands"))
it: obj
user > (if-match (property :name "teun" :city "leiden") obj true)
it: true
user > (if-match (property :name (type 'string)) obj true)
it: true
user > (if-match (property :city (regex "lei") name) obj name)
it: "leiden"
